# AIGC 自动化内容工厂需求与技术实现文档

## 1. 项目背景

内容团队在热点发现、脚本创作、素材制作、审核发布等环节通常依赖人工串联，问题主要在于：

1. 热点抓取分散，难以形成统一候选池
2. 选题和脚本质量依赖个体经验，难复用
3. 图片、音频、字幕、视频产出步骤分裂，失败不可追踪
4. 审核与发布链路缺少系统支持，自动化程度低
5. 多平台运营缺少统一任务台和资产沉淀能力

本项目目标是在企业内部构建一套可运行、可扩展、可审计的 AIGC 内容生产系统。

## 2. 建设目标

### 2.1 业务目标

1. 实时聚合多平台热点，形成统一热点池
2. 基于账号定位和平台偏好做 AI 选题决策
3. 自动生成脚本、图片提示词、音频、字幕和视频资产
4. 支持审核通过后的多平台发布
5. 对内容任务、媒体资产、发布结果进行全链路记录

### 2.2 一期交付目标

一期优先交付最小完整闭环，不追求直接覆盖所有外部生产能力。

一期必须完成：

1. 热点抓取与热点入库
2. 选题决策与脚本生成
3. 任务流转状态机
4. 媒体资产真实生成与展示
5. 审核、驳回、发布接口
6. 可操作的前端管理台
7. 本地安全配置加载

### 2.3 非目标

1. 不在一期内承诺所有平台真实自动发布
2. 不在一期内实现复杂数据归因与效果回流
3. 不在一期内默认实现绕过平台限制的抓取或登录逻辑

## 3. 关键假设

以下前提未在当前环境中验证，设计时按最合理方式预留扩展点：

1. 小红书、抖音、B 站真实发布能力需以平台官方开放能力、企业资质或稳定登录态方案为前提
2. 图片、TTS、视频合成等能力可能依赖独立运行节点或第三方服务
3. 真实大模型调用需要配置可用的模型网关与 API Key

因此当前项目采取“真实接口抽象 + 默认本地模拟实现”的一期策略。

## 4. 用户角色

1. 运营人员：创建内容任务、查看热点、审核内容、触发发布
2. 审核人员：复核选题与脚本，决定通过或驳回
3. 开发/运维人员：配置数据源、模型参数、发布适配器和监控

## 5. 核心业务流程

### 5.1 输入

1. 热点数据源配置
2. 账号定位和任务名称
3. 目标发布平台
4. 审核意见
5. 发布动作
6. 本地模型与本地发布配置

### 5.2 处理流程

1. 系统抓取或读取最新热点
2. AI 门面从热点池中选择候选选题
3. 系统为选题生成结构化脚本
4. 系统为任务生成图片、音频、字幕、视频等资产文件与记录
5. 任务进入审核态
6. 审核通过后进入发布态
7. 发布结果回写任务与发布记录表

### 5.3 状态变化

任务状态机：

`INIT -> HOTSPOT_FETCHED -> TOPIC_SELECTED -> VIDEO_READY -> REVIEW_PENDING -> REVIEW_APPROVED -> PUBLISHING -> PUBLISHED`

异常状态：

`REVIEW_REJECTED`

### 5.4 输出

1. 热点记录
2. 结构化选题候选
3. 结构化脚本
4. 媒体资产文件与元数据
5. 平台发布记录
6. 仪表盘与任务详情页

## 6. 功能需求

### 6.1 热点管理

系统应支持：

1. 手动触发热点采集
2. 查询最近热点列表
3. 对热点按时间和热度排序
4. 将热点统一转为标准结构后落库

标准字段包括：

1. 来源平台
2. 来源话题 ID
3. 标题
4. 摘要
5. 热度分
6. 抓取时间
7. 原始载荷

### 6.2 内容任务创建

系统应支持：

1. 输入任务名称
2. 输入账号定位
3. 选择目标平台
4. 可选输入偏好选题关键词
5. 创建后自动触发工作流

### 6.3 选题决策

系统应支持：

1. 从热点列表中选出最多 3 个候选题
2. 为每个候选题输出推荐理由
3. 输出目标受众、建议平台、风险提示
4. 将最终选中话题写入任务主表

### 6.4 脚本生成

系统应支持输出结构化脚本：

1. 标题
2. 开场钩子
3. 分段内容
4. 结尾 CTA
5. 标签
6. 预计时长
7. 音色/语气建议
8. 配图提示词

### 6.5 媒体资产

一期要求系统在本地直接生成真实资产文件，并同步写入资产记录。

资产类型包括：

1. 封面图
2. 音频
3. 字幕
4. 视频

每类资产至少记录：

1. 任务 ID
2. 资产类型
3. 文件名
4. 文件路径或 URL
5. 状态
6. 参数摘要

### 6.5.1 本地免费素材链路

系统在无外部付费服务前提下应能完成以下链路：

1. 使用 Java2D 生成封面图
2. 使用 macOS `say` 生成中文语音
3. 生成独立 `srt` 字幕文件
4. 使用 `ffmpeg` 合成 `mp4` 视频
5. 通过 HTTP 暴露已生成文件

### 6.6 审核流程

系统应支持：

1. 查看任务详情和脚本内容
2. 审核通过
3. 审核驳回
4. 写入审核意见

### 6.7 发布流程

系统应支持：

1. 审核通过后发布
2. 支持传入发布平台列表
3. 按平台生成发布记录
4. 回写平台内容 ID、状态和消息

一期默认提供本地模拟发布器，并保留真实平台发布适配器接口。

### 6.8 前端管理台

前端应支持：

1. 仪表盘概览
2. 热点列表
3. 创建任务
4. 任务列表
5. 任务详情
6. 审核通过/驳回
7. 发布操作

### 6.9 本地配置安全加载

系统应支持以下本地配置加载方式：

1. `.env`
2. `application-local.yml`

要求：

1. 本地配置文件不提交到版本库
2. 能覆盖 AI 网关地址、模型名、API Key、端口和发布配置
3. 支持开发者本机直接切换 OpenAI、千问兼容网关或其他兼容实现

## 7. 技术选型

### 7.1 后端

1. Java 21
2. Spring Boot 3.4.x
3. MyBatis-Plus
4. LangChain4j
5. H2
6. Actuator
7. FFmpeg
8. Playwright Chromium

### 7.2 前端

一期采用内嵌式静态前端：

1. HTML
2. CSS
3. Vanilla JavaScript

这样可以在当前环境下零额外编译成本完成闭环演示；后续若需要独立前后端分离，可迁移到 React/Vue 项目。

## 8. 系统架构

### 8.1 分层设计

1. Controller：提供 REST API
2. Service：承载工作流与业务逻辑
3. Provider/Publisher：承载热点采集与平台发布适配能力
4. Mapper：MyBatis-Plus 数据访问
5. Static UI：管理后台
6. Scripts：本地浏览器自动化辅助脚本

### 8.2 关键设计原则

1. 主流程与外部依赖解耦
2. 任务过程状态可追踪
3. 脚本与资产采用结构化数据
4. 先保证流程闭合，再逐步替换为真实能力
5. 对平台页面结构变化保持最小耦合，优先使用可配置地址和通用填充策略

## 9. 数据模型

当前实现包含以下表：

1. `hotspot_record`
2. `content_task`
3. `topic_candidate`
4. `content_script`
5. `media_asset`
6. `publish_record`

## 10. API 需求

### 10.1 仪表盘

1. `GET /api/dashboard/summary`

### 10.2 热点

1. `GET /api/hotspots`
2. `POST /api/hotspots/collect`

### 10.3 任务

1. `POST /api/tasks`
2. `GET /api/tasks`
3. `GET /api/tasks/{id}`
4. `POST /api/tasks/{id}/approve`
5. `POST /api/tasks/{id}/reject`
6. `POST /api/tasks/{id}/publish`

## 11. 安全与合规要求

1. 不在配置文件中明文写生产账号密码
2. 模型 API Key 使用环境变量注入
3. 平台抓取与发布应遵循官方协议和风控要求
4. 自动发布前必须保留人工审核开关

## 12. 稳定性要求

系统必须具备：

1. 参数校验
2. 全局异常处理
3. 任务状态可回溯
4. 外部能力可替换
5. 管理台可直接操作完整流程
6. 本地配置与本地凭证加载不污染仓库

## 13. 验收标准

满足以下条件视为一期达成：

1. 项目可在本地单命令启动
2. 用户能通过前端创建任务
3. 系统能自动生成任务详情、脚本以及可访问的本地素材文件
4. 用户能通过前端审核通过或驳回
5. 审核通过后可触发发布并查看发布记录
6. H2 中可查询热点、任务、脚本、资产和发布数据
7. 本地配置文件可在不改仓库代码的情况下接入真实模型网关

## 14. 当前实现与需求对照

已完成：

1. 管理台与核心 API
2. 任务状态流转
3. MyBatis-Plus 落库
4. LangChain4j 接入入口
5. 本地真实封面、语音、字幕、视频生成
6. 本地模拟热点与发布适配器
7. B站扫码登录后自动填充视频、标题、简介

待替换为真实生产能力：

1. 真实热点源接入
2. 真实模型结构化输出
3. Stable Diffusion / ComfyUI 图片生成
4. 抖音/小红书扫码后自动填充
5. 多平台官方发布适配
